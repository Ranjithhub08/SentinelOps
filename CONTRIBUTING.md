# Contributing to SentinelOps

First off, thank you for considering contributing to SentinelOps! We love seeing our community get involved in making the platform even better.

## How Can I Contribute?

### Reporting Bugs
If you find a bug, please open an issue on GitHub. Include:
- A clear, descriptive title.
- Steps to reproduce the bug.
- A description of the expected vs. actual behavior.
- Screenshots if applicable.

### Suggesting Enhancements
We’re always looking for new ideas! To suggest an enhancement:
- Check if it’s already been suggested in the issues.
- Open a new issue with the label `enhancement`.
- Explain how the feature would work and why it would be useful.

### Pull Requests
1.  **Fork the repository** and create your branch from `main`.
2.  **Follow the coding style** of the existing project (PEP8 for Python, standard Next.js patterns for Frontend).
3.  **Ensure it builds**: Run the `start_test_env.sh` script to verify your changes don't break the system.
4.  **Issue a Pull Request**: Provide a detailed description of your changes.

## Development Setup

See the [Running Locally](README.md#running-locally) section in the main README for full setup instructions.

### Branch Naming Convention
- `feat/feature-name`
- `fix/bug-name`
- `docs/documentation-update`
